# Control de calidad mínimo de SVperitus

Este bloque reúne los instrumentos mínimos de control interno de `SVperitus`.

Su finalidad es dejar trazabilidad local, suficiente y sobria del estado de agentes, fases, artefactos, cierres y deuda del repositorio, sin duplicar la logística técnica de `SV-lenguaje-de-computacion` ni desplazar la autoridad doctrinal de `SV-matematica-semantica`.

## Distribución entre sedes

- `SV-matematica-semantica`: autoridad doctrinal superior y orientación global.
- `SV-lenguaje-de-computacion`: sede logística fuerte del control técnico y de la trazabilidad operativa.
- `SVperitus`: adaptación aplicada y ligera, suficiente para sostener agentes, fases, artefactos y deuda local.

## Reglas de uso

1. Este bloque sólo fija alcance, hitos, deuda y actuaciones relevantes del propio repositorio.
2. Toda afirmación estructural debe apoyarse en evidencia material suficiente.
3. Los frentes que afecten a DSL, validator, lowering, serialización, consulta, runner o backend deben comparecer también en `SV-lenguaje-de-computacion/docs/calidad/`.
4. La armonía entre sedes exige coherencia, no simetría exacta.

## Nota final

Este bloque es deliberadamente mínimo. Su función no es inflar burocracia, sino dejar constancia suficiente dentro de `SVperitus` para facilitar continuidad, lectura pública y control local aplicado.
